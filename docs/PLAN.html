MRIViewer — Implementation Plan

An open-source, strictly local web app: drop a folder of MRI DICOM files, get a fully interactive, GPU-raymarched 3D volume with linked MPR slice views. Built on React Three Fiber over WebGL2. Zero network calls at runtime — imaging data never leaves the machine.

React Three Fiber 9.6 three r185 · WebGL2 dicom-parser + WASM codecs Vite 8 · TypeScript strict MIT · local-only
Written for agentic execution (Claude Code, ultracode). Every decision below is locked with rationale; every phase has objective acceptance criteria and verification commands. Provenance: 13 research agents across 6 topics; 122 load-bearing claims adversarially fact-checked against npm, GitHub source, MDN, Khronos and DICOM PS3.x on 2026-07-03. 1 · Goal, scope, non-goals

In scope (v1)

  • Folder ingest: drag-and-drop and folder picker; hundreds to ~2000 files; multiple studies/series per folder with a series browser.
  • Classic single-frame MR series → 3D volume; Enhanced (multi-frame) MR in Phase 5.
  • Compressed transfer syntaxes via self-hosted WASM codecs (JPEG Lossless, JPEG-LS, JPEG 2000, RLE, HTJ2K, deflated).
  • 3D view: raymarched DVR with editable transfer function, MIP, shaded ISO; perspective + orthographic; clip box; orientation labels & gizmo.
  • MPR: axial/sagittal/coronal panes with linked crosshairs, wheel scrub, thick-slab MIP — all sampling one shared 3D texture.
  • Window/level (radiology mouse conventions), percentile auto-window, colormaps, PNG export, metadata panel, hotkeys.
  • Hard local-only guarantee: strict CSP, zero-network CI test, all assets bundled.

Non-goals (v1)

  • No diagnosis: ship a visible “not a medical device — not for diagnostic use” disclaimer.
  • No DICOMweb/PACS networking, no server component, no accounts, no telemetry — ever.
  • No segmentation, registration, fusion, or quantitative maps (ADC scaling via Philips private tags explicitly out).
  • No mobile/Safari support (folder ingest is impractical there); documented as desktop-only.
  • No WebGPU (R3F v10 is alpha-only); architecture avoids deep coupling to ease later migration.
  • No PWA/service worker and no OPFS persistence by default (PHI-on-disk trade-off; see Privacy).

Naming note: EPAM’s dormant-ish epam/mriviewer (“MRI Viewer”, Apache-2.0) already exists. We keep the name — different org, different stack — but the README should say “not affiliated with EPAM MRI Viewer” for discoverability.

5 · DICOM → volume pipeline (the correctness core)

5.1 Folder ingest

  • Dual-path drop handler: in the drop tick, synchronously map all dataTransfer.items → getAsFileSystemHandle?.() ?? webkitGetAsEntry(), then Promise.all. Any await before mapping makes later items resolve null (MDN-documented).
  • readEntries() returns ≤100 entries per call in Chromium — loop until an empty batch or large studies silently lose slices.
  • Fallback button: <input webkitdirectory multiple> (Baseline since 2025-08; all desktop browsers). Optional Chromium nicety: showDirectoryPicker() — feature-detect; Firefox/Safari have formally rejected it.
  • No accept= attribute, no extension filtering — DICOM files often have none. Identify by "DICM" magic at byte 128 in the worker; skip DICOMDIR, reports, secondary captures (filter SOPClassUID: keep MR Image Storage 1.2.840.10008.5.1.4.1.1.4 and Enhanced MR …4.1).
  • Normalize entry.fullPath (drop) vs file.webkitRelativePath (input) into one internal path scheme.

5.2 Worker pool

  • Pool of max(1, ⌊hardwareConcurrency/2⌋) module workers (Cornerstone’s production default), created as new Worker(new URL('./ingest.worker.ts', import.meta.url), {type:'module'}) — URL constructor literally inline or Vite won’t bundle it.
  • Post File objects in (structured-cloneable, lazy — bytes are read in the worker); transfer decoded pixel ArrayBuffers back (zero-copy; cloning a 32 MB buffer ≈ 300 ms vs 6.6 ms transferred).
  • Per-file try/catch isolation: one malformed file never kills the ingest. Sanity-cap element lengths against file size to defeat allocation bombs from corrupt headers.
  • New folder drop cancels the in-flight ingest (generation counter; workers check and abandon stale jobs).

5.3 Grouping & splitting (a SeriesInstanceUID is not a volume)

  1. Group by StudyInstanceUID → SeriesInstanceUID (0020,000E).
  2. Split further by distinct ImageOrientationPatient (localizers/scouts mix orientations in one series).
  3. Detect duplicate projected positions → split by 4D tags: EchoNumbers (0018,0086), TemporalPositionIdentifier, DiffusionBValue (0018,9087), TriggerTime, ImageType (magnitude/phase). One “volume candidate” per group; the series browser lists candidates.

5.4 Geometric sort — the only correct ordering

Never sort by InstanceNumber, SliceLocation, or filename — acquisition order is interleaved/descending/vendor-random (the InstanceNumber sort in ITK’s DCMTK reader is a documented scrambled-volume bug). This is the GDCM/nibabel/SPM/Clunie algorithm.

5.6 Pixel & intensity pipeline (MR has no Hounsfield units)

  1. Decode per TransferSyntaxUID. Encapsulated frames: readEncapsulatedImageFrame + createJPEGBasicOffsetTable (fragments-per-frame is not 1:1). Uncompressed: view into the parse buffer using BitsAllocated/PixelRepresentation.
  2. Sign-extend when PixelRepresentation=1 and BitsStored<BitsAllocated (12-in-16 is common): v = (raw << (16-bits)) >> (16-bits). Mask above HighBit for unsigned legacy files. Wrong handling = salt-and-pepper wraparound in bright regions.
  3. Rescale: v·RescaleSlope + RescaleIntercept when present (defaults 1/0 — the GE/Siemens norm). Do not chase Philips private (2005,100E) scaling — viewer, not quantimetry.
  4. MONOCHROME1 → invert.
  5. Normalize per-volume with robust percentiles (p0.5–p99.5 histogram clip, Slicer-style) → Float16 [0,1]. Seed initial window from WindowCenter/Width (0028,1050/1051) if present (may be multi-valued/per-slice/absent — first value, optional preset only).
  6. PixelSpacing order trap: (0028,0030) is row spacing \ column spacing (vertical first). Backslash-split multi-valued DS strings — never parseFloat the whole string.
  7. After assembly, drop all per-slice buffer references (keep the cheap File handles for lossless re-parse) — raw buffers + volume + GPU copy can transiently triple memory.

6 · Coordinate systems — the anti-mirroring spec

Highest-severity bug class in this project: a left/right-mirrored brain renders 100% plausibly and passes every eyeball test. This section is normative; Phase 1 encodes it in tests against an asymmetric phantom.

Definitions

  • Patient space = DICOM LPS (+X Left, +Y Posterior, +Z Superior), in mm.
  • Voxel→LPS affine (Image Plane Module C.7.6.2): columns are X̂ = row-cosines·colSpacing, Ŷ = col-cosines·rowSpacing, Ẑ = n̂·zSpacing where n̂ = row×col and zSpacing = median consecutive projected-IPP delta; translation = IPP of the first sorted slice (center of first voxel).
  • World space = RAS, Z-up: worldFromLPS = diag(-1,-1,1). So world +X = patient Right, +Y = Anterior, +Z = Superior. Set camera.up=(0,0,1).
  • Texture space: UVW = voxel index / dimension; W increases with the sorted slice index (i.e. along +n̂). The mesh gets modelMatrix = worldFromLPS · lpsFromVoxel · scale(1/dims) so shader marching happens in unit-texture space with zero per-step transforms.

Consequences (each becomes a test)

  • Slice-stacking consistency: sorting ascending by dot(n̂, IPP) with n̂ = row×col guarantees W and Ẑ agree — no mirror through the slice plane.
  • Orientation labels: for each pane edge, take the edge’s world direction, find the dominant patient axis from the direction cosines, label A/P/L/R/S/I (threshold 0.5). Shown on all four edges of every MPR pane and as a 3D gizmo.
  • Radiological convention: axial pane displays patient-left on screen-right by default (toggle to neurological).
  • Anisotropy: mesh scaled by (nx·sx, ny·sy, nz·sz) mm — a 0.9×0.9×5 mm acquisition must render visibly slab-like, never cubified.
  • Measurements transform endpoints through the affine to patient mm before Euclidean distance — never pixel-count × one spacing value.
Color fidelity: R3F’s Canvas defaults to ACES filmic tone mapping + sRGB output, which silently distorts grayscale ramps. Use <Canvas flat linear> (NoToneMapping, linear), keep the volume texture and LUT in linear space, and add a readback test asserting a known intensity maps to the exact LUT color.

7 · Rendering spec

7.1 Texture setup

  • new THREE.Data3DTexture(halfFloatData, nx, ny, nz); format=RedFormat, type=HalfFloatType (→R16F), minFilter=magFilter=LinearFilter (defaults are Nearest — the #1 “blocky volume” bug), generateMipmaps=false, unpackAlignment=1, needsUpdate=true. Data = Uint16Array of half-float bit patterns via DataUtils.toHalfFloat — never raw integer intensities.
  • Box mesh rendered with side: THREE.BackSide so the camera can enter the volume; clamp t.x = max(t.x, 0) for the inside-the-box case.

7.2 Raymarch fragment shader (per-mode contract)

  • Precision: precision highp float; precision highp sampler3D; — three’s reference uses mediump, which quantizes 16-bit data on some GPUs.
  • Step size: dt = min over axes of 1/(dims·|dir|) × quality factor; loop cap ≈ √3 × maxDim (887 for 512³).
  • Jitter from day one: offset ray start by a fraction of one step using interleaved-gradient-noise on gl_FragCoord — fixed-step marching without it produces wood-grain banding that adaptive stepping alone cannot fix.
  • MIP must traverse the full ray (no early-out) + 4-substep refinement around the max. DVR/ISO early-terminate at α ≥ 0.95. ISO: threshold pre-test, half-step-back refinement, central-difference gradient (6 samples, only at the hit), Blinn-Phong, normal flipped toward viewer.
  • Transfer function: 256×1 RGBA DataTexture LUT sampled at the window-normalized value (v−clim.x)/(clim.y−clim.x) with half-texel correction. Presets: gray, viridis, hot-iron. u_clim is window/level.
  • Clipping: six-plane clip box applied in-shader by shrinking the ray’s [tmin,tmax] analytically — three’s clippingPlanes only clip the box surface, not the marched interior.
  • Interaction quality scaling: on controls start → halve dpr/double dt; on end → restore + invalidate(). Early termination + skip-lighting-below-threshold for DVR.

7.3 R3F integration rules

  • <Canvas frameloop="demand" flat linear gl={{antialias:false, powerPreference:'high-performance'}}>. Every imperative mutation pairs with invalidate(); drei controls auto-invalidate.
  • Uniform updates: mutate matRef.current.uniforms.X.value (fiber ≥9.6 merges incoming uniforms props into a stable object — mutating your original object is dead code). Never setState in useFrame/pointermove; route fast values through zustand transient subscriptions.
  • Disposal: textures living only in material.uniforms are invisible to R3F’s auto-dispose — on study swap call texture.dispose() in effect cleanup and assert gl.info.memory.textures returns to baseline (a dev-mode test). StrictMode double-mount makes non-idempotent texture creation double GPU memory in dev.
  • Suspense: parse is keyed by suspend(parseStudy, [seriesKey]); call clear([seriesKey]) on close. Fallback inside Canvas must be <Html>, not DOM.

7.4 MPR panes

  • Quad layout via drei <View frames={1}> ×4 over one absolutely-positioned fullscreen Canvas (eventSource = grid container). All views share the frameloop — acceptable; slice shaders are cheap.
  • Each 2D pane: orthographic camera + fullpane quad whose fragment shader samples the same sampler3D at a fixed texture coordinate along its axis; wheel scrubs the slice index; optional thick-slab MIP over N samples along the normal.
  • Linked crosshairs: one shared patient-mm point in zustand; panes project it via the affine; clicking any pane updates it (transient sub → uniforms → invalidate).

9 · Privacy & local-only enforcement

“Local-only” is a tested property, not a README promise. Three independent layers:

1 · Build-time

base:'./', all fonts/assets bundled (system font stack), no CDN references. CI greps dist/ for https?:// and fails on any hit (sourcemap references excluded).

2 · Runtime CSP

Production-only <meta http-equiv="Content-Security-Policy"> injected by a build-mode transformIndexHtml plugin (dev needs the HMR websocket).

3 · CI proof

Playwright routes **/*, aborts + records any non-localhost request, ingests a fixture study, renders, asserts zero violations.

  • 'wasm-unsafe-eval' is required for the codec WASM; worker-src 'self' blob: because bundlers sometimes load workers from blob URLs. Meta-CSP can’t express frame-ancestors — note in SECURITY.md, don’t claim it.
  • PHI is real even locally: headers carry PatientName/ID/BirthDate/StudyDate/institution (PS3.15 Annex E is the de-identification reference if an anonymized-export feature lands later). No logging of header values; no persistence by default; OPFS caching only ever as explicit opt-in of the derived volume with a visible clear control — not in v1.
  • itk-wasm (dev oracle) and dicom-codec resolve WASM from CDNs by default — the app bundle must self-host all codec WASM (Vite asset handling + explicit URL config); the dist-grep catches regressions.

10 · Test strategy — how an autonomous agent proves correctness

Keystone: the synthetic phantom generator (tools/make-phantom.ts, dcmjs-based, committed fixtures). An asymmetric phantom — a bright marker in the patient Left-Anterior-Superior octant plus distinct X/Y/Z gradient ramps, anisotropic 1×1×2.5 mm spacing — makes orientation, mirroring, spacing and sorting machine-checkable. This single tool unblocks unit tests, e2e, the orientation guarantee, and the zero-network constraint (fixtures live in-repo).

13 · Pitfall checklist (pre-commit review lens for the implementing agent)

DICOM

  • ☐ Sort = dot(row×col, IPP). Never InstanceNumber/SliceLocation/filename.
  • ☐ zSpacing from consecutive projected IPP deltas — never SliceThickness; SpacingBetweenSlices is Type-3, sometimes wrong, legally negative.
  • ☐ PixelSpacing = row\column (vertical first); backslash-split all multi-valued DS.
  • ☐ Sign-extend 12-in-16 signed; mask above HighBit unsigned; MONOCHROME1 invert.
  • ☐ Enhanced MR: geometry lives in functional groups, not top-level tags.
  • ☐ Implicit VR is the default syntax — never assume explicit. Deflated needs pako pre-parse.
  • ☐ Encapsulated frames via readEncapsulatedImageFrame + offset-table helper.
  • ☐ HTJ2K UIDs are 1.2.840.10008.1.2.4.201/.202/.203 (upstream README has a typo).

WebGL / three.js

  • ☐ LinearFilter set explicitly (Data3DTexture defaults to Nearest); unpackAlignment=1; no mipmaps.
  • ☐ R16F via toHalfFloat bit patterns; never integer textures; R32F only after checking OES_texture_float_linear.
  • ☐ BackSide box; clamp t.x≥0; highp sampler3D.
  • ☐ Jittered ray start from day one; MIP never early-exits; premultiplied front-to-back compositing only.
  • ☐ Clipping in-shader (t-range), not material.clippingPlanes.
  • ☐ Anisotropic scale from real spacing or anatomy squashes.

R3F

  • ☐ No setState in useFrame/fast pointer events; transient zustand subs; every mutation + invalidate().
  • ☐ Mutate matRef.current.uniforms.X.value (v9.6 merge semantics).
  • ☐ Dispose uniform-held textures on swap; StrictMode-idempotent creation; assert gl.info.memory.
  • ☐ One canvas + View for MPR; frames={1}; eventSource on the container.
  • ☐ Canvas flat linear; demand-mode animations start inside rAF after invalidate().
  • ☐ Suspense fallback inside Canvas = <Html>, never DOM.

Ingest / build / privacy

  • ☐ Drop-tick synchronous handle mapping; readEntries 100-batch loop; Firefox callback promisified.
  • ☐ Files→workers (lazy), buffers back via transfer list; release per-slice buffers post-assembly.
  • ☐ Worker URL literal-inline for Vite; smoke-test workers under base:'./' from a nested path.
  • ☐ CSP prod-only; wasm-unsafe-eval + worker-src blob: present; file:// documented as unsupported.
  • ☐ No extension filtering; DICM sniff at byte 128; skip DICOMDIR/SC/SR.
  • ☐ Never log header values; no OPFS by default.
